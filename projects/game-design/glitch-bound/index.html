Technical Details (Game Overview)

Click to view technical game details

GlitchBound is a 2D puzzle-platformer where the player uses in-game glitches to navigate and manipulate a fractured simulation. Designed with a retro-futuristic pixel art aesthetic and immersive sound design, the game challenges players to exploit intentional bugs—such as wall clipping, gravity shifting, object duplication, and time glitches—to solve puzzles and recover lost memory fragments.

Target Audience

Players aged 12 and up who enjoy sci-fi puzzle-platformers with unique mechanics and deep lore. Ideal for fans of Celeste, Fez, The Messenger, and Katana Zero.

Main Selling Points

  • Reality-bending mechanics like Wall Clipping, Gravity Shift, and Time Rewind
  • A Memory Fragment system that ties gameplay to narrative discovery
  • Highly stylized glitch visual effects
  • Layered level design encouraging exploration and creative thinking
  • A gradually revealed story with emotional depth and a cinematic finale

Inspirational Games

  • Celeste – precision platforming + emotional narrative
  • Axiom Verge – glitch-based mechanics and retro vibes
  • Fez – hidden lore and puzzle-focused design
  • Hyper Light Drifter – worldbuilding through visuals and fragments

Technical Specifications

Estimated Total Playtime: 20–35 minutes for a full playthrough, depending on puzzle-solving speed and memory collection.

Performance Metrics
  • Stable framerate on typical desktop PCs running Unity (60 FPS target)
  • Average scene load time: < 3 seconds
  • Memory system tracks and stores up to 20 memory fragments using PlayerPrefs
  • Visual effects optimized through URP with post-processing Volumes

SWOT Analysis

Strengths
  • Unique gameplay mechanic: using glitches intentionally as tools rather than bugs
  • Engaging narrative delivered through collectible memory fragments
  • Stylish retro pixel art with immersive glitch effects
  • Clever level design encourages creative problem solving
  • Modular level structure allows for easy scalability and updates
  • Cross-level progression via memory logs creates replay incentive
  • Puzzle variety grows steadily, avoiding early repetitiveness
Weaknesses
  • No traditional combat may limit appeal for certain player types
  • High reliance on glitch effects may strain some players' eyes
  • Limited NPC or character interaction could impact emotional engagement
  • Puzzle complexity may spike without strong visual or tutorial cues
  • PlayerPrefs save system is vulnerable to resets or accidental clears
  • Lack of dynamic audio/music transitions could weaken atmosphere
  • Some effects (e.g., Time Freeze) require precise tuning for fairness
  • Fixed memory order may reduce freedom for nonlinear players
Opportunities
  • Strong appeal to fans of indie puzzle-platformers
  • Expansion potential: more levels, fragment lore, or combat mechanics
  • Could adapt well to console or handheld platforms
  • Market interest in "meta" or glitch-style games continues to grow
  • Memory system could evolve into a codex or interactive storybook
  • Additional "corrupted" mechanics or worlds could deepen gameplay
  • Visual and sound assets are reusable for sequels or expansions
  • Compelling lore invites possible spin-offs (comics, visual novels, ARGs)
Threats
  • Competes with polished indie hits (Celeste, Fez, etc.)
  • Audience confusion between real bugs and intentional glitch mechanics
  • Scope creep risk: too many mechanics could overwhelm players
  • Needs refined onboarding/tutorial to avoid early frustration
  • Over-scoping mechanics could risk unfinished or unpolished systems
  • Visual simplicity could be misjudged as under-designed
  • Bug fixing is critical—unintended glitches could break immersion
  • Longer-form games may outshine it if narrative isn't paced effectively

Memory Revealer (C#)
using UnityEngine;
using TMPro;

public class MemoryFragment : MonoBehaviour
{
    public string memoryID;
    [TextArea(3, 6)] public string memoryText;
    public GameObject memoryPanel;
    public TextMeshProUGUI memoryTextUI;
    private bool showingMemory = false;

    void OnTriggerEnter2D(Collider2D other)
    {
        if (other.CompareTag("Player") && !PlayerPrefs.HasKey(memoryID))
        {
            ShowMemory();
        }
    }

    void Update()
    {
        if (showingMemory && Input.GetKeyDown(KeyCode.X))
        {
            HideMemory();
        }
    }

    void ShowMemory()
    {
        memoryTextUI.text = memoryText;
        memoryPanel.SetActive(true);
        Time.timeScale = 0f;
        showingMemory = true;
        PlayerPrefs.SetInt(memoryID, 1);
    }

    void HideMemory()
    {
        memoryPanel.SetActive(false);
        Time.timeScale = 1f;
        showingMemory = false;
        Destroy(gameObject);
    }
}
Glitch Transition (C#)
using UnityEngine;
using UnityEngine.SceneManagement;
using UnityEngine.Rendering;
using URPGlitch.Runtime.AnalogGlitch;
using URPGlitch.Runtime.DigitalGlitch;

public class ExitDoor : MonoBehaviour
{
    public Volume glitchVolume;
    public string nextSceneName;
    public float glitchDuration = 2f;
    private AnalogGlitchVolume analogGlitch;
    private DigitalGlitchVolume digitalGlitch;
    private bool isTriggered = false;

    private void Start()
    {
        if (glitchVolume != null && glitchVolume.profile != null)
        {
            glitchVolume.profile.TryGet(out analogGlitch);
            glitchVolume.profile.TryGet(out digitalGlitch);
        }
    }

    private void OnTriggerEnter2D(Collider2D other)
    {
        if (isTriggered) return;
        if (other.CompareTag("Player"))
        {
            isTriggered = true;
            StartCoroutine(TriggerGlitchTransition());
        }
    }

    private System.Collections.IEnumerator TriggerGlitchTransition()
    {
        if (analogGlitch != null)
        {
            analogGlitch.scanLineJitter.value = 0.8f;
            analogGlitch.verticalJump.value = 0.5f;
            analogGlitch.horizontalShake.value = 0.6f;
            analogGlitch.colorDrift.value = 0.9f;
        }
        
        if (digitalGlitch != null)
        {
            digitalGlitch.intensity.value = 0.85f;
        }

        yield return new WaitForSeconds(glitchDuration);
        SceneManager.LoadScene(nextSceneName);
    }
}
